# Discover 页面实现说明

## 功能概述

根据图片设计，我已经成功实现了 discover 页面，包含以下主要功能：

### 1. 地图功能

- 使用现有的 Map 组件（基于高德地图）
- 支持显示活动位置标记点
- 点击标记点可查看活动信息气泡
- 地图与活动列表联动

### 2. 搜索和筛选功能

- 地址搜索输入框
- 多维度筛选：
    - 类别筛选（环保、教育、文化、手工、体验等）
    - 时间筛选（今天、明天、本周、本月）
    - 距离筛选（1km内、3km内、5km内、10km内）
    - 人数筛选（10人以下、10-50人、50-100人、100人以上）

### 3. 活动列表

- 响应式卡片布局
- 显示活动图片、标题、内容、时间、地点、发布人、奖励等信息
- 参与人数和距离显示
- 点击卡片可进入活动详情页

### 4. 活动详情页

- 完整的活动信息展示
- 活动图片、基本信息、详细描述
- 参与要求和标签
- 活动地点地图
- 参加活动按钮

### 5. 导航系统

- 全局导航栏，包含首页、发现、活动、我的等导航项
- 品牌 Logo 和发布活动按钮
- 响应式设计

## 技术实现

### 使用的技术栈

- React 19 + TypeScript
- React Router DOM 用于路由管理
- Tailwind CSS 用于样式设计
- 高德地图 API 用于地图功能

### 主要组件

1. **DiscoverPage** (`src/pages/discover/index.tsx`)
    - 主页面组件
    - 包含地图、搜索、筛选、活动列表

2. **ActivityDetailPage** (`src/pages/discover/activity-detail.tsx`)
    - 活动详情页组件
    - 显示完整的活动信息

3. **Map** (`src/components/Map/index.tsx`)
    - 地图组件
    - 支持标记点和信息窗口

4. **Navigation** (`src/components/Navigation/index.tsx`)
    - 全局导航组件

### 数据管理

- 使用 React Hooks 管理状态
- 模拟活动数据（可替换为真实 API）
- 筛选逻辑实时更新活动列表

## 设计特色

### 颜色方案

- 主题色：绿色 (#4ade80) 和黄色 (#eab308)
- 背景色：白色和灰色 (#f5f5f5)
- 符合 30% 主题色 + 70% 白色灰色的设计要求

### 交互设计

- 悬停效果和过渡动画
- 响应式布局，适配不同屏幕尺寸
- 直观的筛选和搜索体验

## 使用方法

1. 启动开发服务器：

    ```bash
    npm run dev
    ```

2. 访问页面：
    - 首页：http://localhost:5173/
    - Discover 页面：http://localhost:5173/discover
    - 活动详情页：http://localhost:5173/discover/activity/1

3. 功能测试：
    - 在地图上点击标记点查看活动信息
    - 使用筛选条件过滤活动
    - 点击活动卡片进入详情页
    - 测试响应式布局
